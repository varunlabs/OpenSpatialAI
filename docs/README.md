# Documentation Index

This folder is organized into current project documentation, final deliverables, and archived draft/reference material.

## Folder Layout

- `project_manual/`: complete technical manual for the Unity project, from setup to Quest output.
- `final_submission/`: current report and paper deliverables grouped by owner/name.
- `archive/`: older drafts, alternate LaTeX sources, and reference exports that should not be treated as the main current documents.

## What To Use First

Start here:

```text
project_manual/00_START_HERE.md
```

Use this for build/setup/project understanding:

```text
project_manual/
```

Use this for current report/paper deliverables:

```text
final_submission/lavanya/
```

Use this only for reference or recovery:

```text
archive/
```

## Rule For Future Files

- Put final report/paper outputs in `final_submission/<person-name>/`.
- Put technical project documentation in `project_manual/`.
- Put old drafts, temporary versions, or superseded exports in `archive/`.
- Do not mix generated exports, old drafts, and current deliverables in the same folder.
