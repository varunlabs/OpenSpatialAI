# Evidence Folder Guide

Use this folder to collect final validation proof for the HarmonyXR adaptive XR proof-of-concept.

Start with:

- `00_EVIDENCE_INDEX.md`
- `EVIDENCE_SUMMARY_20260514.md`

## Folder Usage
- `01_Build_Proof`: headset launch and build/runtime proof notes.
- `02_Live_Sensor_Validation`: gaze/head/hand/body/spatial tracking proof.
- `03_Context_State_Coverage`: main headset evidence video and all four context-state proof.
- `04_Adaptation_Behavior`: adaptive response evidence using the same combined run.
- `05_Stability_Latency`: continuous runtime console evidence and latency/stability notes.
- `06_Final_Demo`: presentation reference using the same full evidence video.
- `07_Processed_Tables_Charts`: final tables/charts extracted from logs.
- `08_Notes_Limitations`: interpretation boundaries, device notes, and QA observations.

## Primary Video Evidence
- `03_Context_State_Coverage/20260514_run01_full-evidence.mp4`

## Console/Log Filters
Use these filters when reviewing evidence:
- `QA_METRICS`: complete runtime QA evidence line.
- `Phase3`: adaptive behavior and QA panel events.
- `STATE`: context-state evidence if visible in logs.

## Naming Convention
Use this pattern:
`YYYYMMDD_run##_short-description.ext`

Examples:
- `20260514_run01_full-evidence.mp4`
- `20260514_run01_console-evidence.txt`
- `20260514_run01_filtered-evidence.txt`
